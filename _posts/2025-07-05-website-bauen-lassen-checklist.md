Ich helfe seit Jahren Unternehmern beim Bau ihrer Website. Und
mir fällt auf: Die meisten haben keine Ahnung, wo sie anfangen
sollen. Sie wissen, dass sie eine Website brauchen, aber was
alles dazu gehört — das wird erst klar, wenn es zu spät ist.

Deshalb diese Checkliste. Sie basiert auf allem, was ich in der
Praxis habe schiefgehen — und gutgehen — sehen. Nutzen Sie sie,
wenn Sie eine Website bauen lassen, egal ob mit mir oder mit
jemand anderem.

## 1. Ziel und Zielgruppe

Bevor auch nur eine Zeile Code geschrieben wird, muss klar sein,
was die Website erreichen soll. Klingt logisch, aber die Hälfte
der Projekte, die ich bekomme, beginnt mit "ich will einfach eine
Website" — ohne weiteres Ziel.

**Checkliste:**
- [ ] Was ist das Hauptziel? (verkaufen, informieren, Anfragen
      generieren, Branding?)
- [ ] Wer ist die Zielgruppe? (Alter, Suchverhalten, Geräte)
- [ ] Was sollen Besucher auf der Website tun? (anrufen,
      Formular ausfüllen, kaufen, lesen?)
- [ ] Was ist die gewünschte Aktion pro Seite? (Call-to-Action)
- [ ] Welche Wettbewerber gibt es und was machen sie gut/schlecht?

**Beispiel:** Ein Klempner möchte vor allem Anrufe. Dann muss die
Telefonnummer prominent auf jeder Seite stehen, nicht versteckt
auf einer Kontaktseite. Ein Onlineshop möchte verkaufen — dann
ist der Bezahlvorgang der kritischste Teil.

## 2. Funktionen

Hier explodieren Budgets oft, weil während des Projekts "schnell"
etwas dazukommt. Erstellen Sie vorab eine Liste.

**Checkliste:**
- [ ] Wie viele Seiten wird die Website haben? (großzügig
      schätzen)
- [ ] Ist Mehrsprachigkeit erforderlich?
- [ ] Soll es einen Blog oder Nachrichtenbereich geben?
- [ ] Kontaktformular — welche Felder?
- [ ] Newsletter-Anmeldung?
- [ ] Onlineshop? Wenn ja: wie viele Produkte, Zahlungen,
      Versand?
- [ ] Buchungssystem oder Kalender?
- [ ] Kunden-Login-Bereich?
- [ ] Social-Media-Integration?
- [ ] Suche auf der Website?
- [ ] PDF-Downloads oder Dokumentenbibliothek?
- [ ] Cookie-Einwilligungshinweis (DSGVO)

Trennen Sie, was **sein muss** (Minimum Viable Product) von
dem, was **später schön wäre**. Das spart Stress und Geld.

## 3. Texte und Inhalte

Viele Projekte verzögern sich, weil die Inhalte nicht rechtzeitig
fertig sind. Eine Website zu bauen ist schnell, Texte zu
schreiben dauert oft viel länger.

**Checkliste:**
- [ ] Wer schreibt die Texte? (Sie selbst, ein Texter, der
      Entwickler?)
- [ ] Sind die Texte vor dem Launch fertig?
- [ ] Gibt es Bilder, Logos und Videos? (eigene Fotos oder
      Stockbilder?)
- [ ] Werden professionelle Fotos gemacht?
- [ ] Gibt es Bewertungen oder Referenzen zum Zeigen?
- [ ] Müssen PDFs oder Dokumente hochgeladen werden?
- [ ] Gibt es eine Datenschutzerklärung und Cookie-Richtlinie?

**Mein Rat:** Beginnen Sie mit dem Schreiben, sobald Sie das
Projekt besprechen, nicht erst wenn die Website fast fertig ist.
Content ist der am meisten unterschätzte Teil jedes
Webprojekts.

## 4. Technik

Hier unterscheiden sich Anbieter enorm. Eine WordPress-Website
ist etwas anderes als eine TYPO3-, Webflow- oder
Custom-Website. Jede Wahl hat Vor- und Nachteile.

**Checkliste:**
- [ ] Welches CMS oder Platform? (WordPress, TYPO3, Webflow,
      Custom?)
- [ ] Wer kümmert sich um Domain und Hosting?
- [ ] Was sind die Hosting-Kosten pro Monat/Jahr?
- [ ] Ist ein SSL-Zertifikat inbegriffen?
- [ ] Wird die Website mobilfreundlich sein?
- [ ] Wie steht es um die Ladegeschwindigkeit? (siehe
      pagespeed.web.dev)
- [ ] Gibt es eine Backup-Regelung?
- [ ] Gibt es eine Staging-Umgebung zum Testen?
- [ ] Wird Google Analytics (oder eine Alternative) installiert?

**Hinweis:** Günstiges Hosting scheint attraktiv, aber eine
langsame Website kostet Sie Kunden. Investieren Sie in
qualitatives Hosting.

## 5. Design

Der Look-and-Feel macht den ersten Eindruck. Aber Design sollte
nicht nur schön sein — es muss funktional sein.

**Checkliste:**
- [ ] Wird ein Design erstellt, bevor gebaut wird?
- [ ] Werden mehrere Designvorschläge gemacht?
- [ ] Kann ich Feedback zum Design geben?
- [ ] Ist das Design responsive (mobil, Tablet, Desktop)?
- [ ] Wird Barrierefreiheit berücksichtigt? (Kontrast,
      Lesbarkeit, Tastaturnavigation)
- [ ] Passt das Design zum Corporate Design? (Farben,
      Schriftarten, Logo)
- [ ] Werden Icons und Bilder professionell verarbeitet?
- [ ] Ist das Design SEO-freundlich? (große Bilder? schlechter
      Kontrast?)

## 6. SEO (Suchmaschinenoptimierung)

SEO ist keine Nebensache, die man später regelt. Die technische
Basis muss während des Baus gelegt werden.

**Checkliste:**
- [ ] Wird die Website technisch SEO-freundlich gebaut?
- [ ] Kommen eindeutige Title-Tags und Meta-Descriptions?
- [ ] Wird eine Sitemap generiert?
- [ ] Wird die Website mit der Google Search Console verknüpft?
- [ ] Werden Bilder optimiert? (Größe, Alt-Texte)
- [ ] Kommt eine robots.txt-Datei?
- [ ] Werden strukturierte Daten (schema.org) hinzugefügt?
- [ ] Ist die Geschwindigkeit der Website optimiert?

Ohne technisches SEO wird es sehr schwer, jemals gefunden zu
werden. Das muss von Tag 1 an stimmen.

## 7. Launch und Übergabe

Dieser Moment wird oft unterschätzt. Die Website ist live — und
dann? Stellen Sie sicher, dass Sie nicht mit leeren Händen
dastehen.

**Checkliste:**
- [ ] Wird die Website vor dem Launch getestet? (alle Browser,
      mobil, Formulare, Links)
- [ ] Bekomme ich eine Einweisung oder Anleitung?
- [ ] Kann ich selbst Texte ändern? (und wenn ja, wie?)
- [ ] Bekomme ich Zugang zum CMS und zum Hosting?
- [ ] Wird die Domain auf meinen Namen registriert?
- [ ] Bekomme ich die Passwörter und Zugangsdaten?
- [ ] Gibt es eine Nachbetreuungszeit (z.B. 1 Monat) für
      Korrekturen?
- [ ] Was kostet Wartung nach dem Launch?
- [ ] Wie schnell wird bei Problemen reagiert?
- [ ] Wird der Quellcode übergeben? (wichtig bei
      Individualentwicklung)

## 8. Budget und Planung

Zum Schluss: das finanzielle Bild. Seien Sie realistisch — eine
gute Website kostet Geld, aber sie bringt auch etwas.

**Checkliste:**
- [ ] Gibt es ein klares Angebot mit Spezifikationen?
- [ ] Was ist NICHT inbegriffen? (Texte, Fotos,
      Zusatzfunktionen)
- [ ] Wird in Phasen mit Meilensteinen gearbeitet?
- [ ] Wie ist die Zahlungsregelung? (z.B. 30/30/40)
- [ ] Was passiert bei Verzögerungen? (von beiden Seiten)
- [ ] Sind die Hosting- und Wartungskosten vorab klar?
- [ ] Werden Nachträge und Änderungen separat berechnet? (wie
      und zu welchem Satz?)

## Praxisbeispiel: von vager Anfrage zu klarem Plan

Ein Kunde kam mit "ich möchte eine neue Website" zu mir. Nach
Nachfragen stellte sich heraus: Sie hatten einen
Produktkatalog mit 500+ Artikeln, Mehrsprachigkeit (NL/EN/DE),
einen Blog und ein Kontaktformular, das direkt an ihr CRM
ankoppeln sollte. Außerdem sollte das Team selbst Produkte
hinzufügen können.

Das ist keine "einfach eine Website" — das ist ein Projekt mit
einem klaren Plan, Budget und Zeitplan.

Die obige Checkliste half uns, alle Anforderungen auf den Tisch
zu bekommen, bevor eine Zeile Code geschrieben wurde. Das
Projekt lief reibungslos, im Budget und wurde pünktlich
abgeliefert.

## Zum Schluss

Eine Website bauen zu lassen ist eine Investition. Diese
Checkliste hilft Ihnen, die häufigsten Fehler zu vermeiden:
unklare Ziele, vergessene Funktionen, zu spätes Beginnen mit
dem Content und Überraschungen im Nachhinein.

Drucken Sie sie aus, besprechen Sie sie mit Ihrem Entwickler,
und stellen Sie sicher, dass Sie Antworten auf alle Punkte
haben. Das spart Ihnen Kopfschmerzen, Zeit und Geld.
